## 2️⃣ AS-GCN 데이터 요구량 분석

### AS-GCN 개요

**ST-GCN 대비 개선점:**
```
1. A-link Inference Module
   - Encoder-Decoder 구조
   - Action-specific latent dependencies 학습
   - 동적 그래프 생성

2. Structural Links
   - Higher-order dependencies (2-hop, 3-hop)
   - 더 먼 관절 간 관계 모델링

3. Future Pose Prediction
   - Self-supervision
   - 더 세밀한 action pattern 학습
```

## 6️⃣ 교훈

### 기술적 교훈

**1. 데이터 > 알고리즘**
```
최신 알고리즘(ST-GCN, AS-GCN) < 적절한 데이터 + 기본 알고리즘(RF)

→ 데이터 품질과 양이 가장 중요
→ 알고리즘은 데이터가 충분할 때 의미
```

**2. 증강의 한계**
```
증강 ≠ 새로운 데이터
- 단순 변형만 가능
- 다양성 추가 불가
- 오히려 해로울 수 있음

→ 실제 데이터 수집이 정답
```

**3. Val ≠ Test**
```
Val 성능이 높아도:
- Test 성능 보장 안됨
- Overfitting 가능
- 일반화 능력 중요

→ 충분한 데이터가 해법
```

### 프로젝트 관리 교훈

**1. 실용성 우선**
```
최신 기술 < 검증된 기술 (안정성)
복잡한 모델 < 간단한 모델 (유지보수)
논문 성능 < 실전 성능 (일관성)

→ Random Forest가 최선
```

**2. 단계적 접근**
```
Phase 1 → Phase 2 → Phase 3
- 각 단계 검증
- 결과 보고 결정
- 무리한 투자 방지

→ 위험 관리
```

**3. ROI 고려**
```
투자: 950만원 + 18개월
수익: 성능 ±2%

→ ROI 낮음
→ 투자 불필요
```
