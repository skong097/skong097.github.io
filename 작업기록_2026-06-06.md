**파일**: `content/about/index.md`

진행 경위(반복 수정):
1. 갤러리 단일 항목이 좌측 320px로 좁게 표시 → `auto-fill` → `auto-fit` 변경.
   - 단일 항목일 때 `auto-fill`은 빈 트랙(320px)을 만들어 항목이 묶임. `auto-fit`은 빈 트랙을 접어 페이지 폭 전체로 확장.
   - "반영 안 됨" 증상은 **브라우저 캐시**(CSS가 HTML 인라인이라 옛 HTML 캐싱) → 강력 새로고침으로 해결.
2. 최종 요구: 갤러리가 **복수**가 될 예정 → **3×N 카드 그리드로 원복**(데스크톱 3열 / 태블릿 2열 / 모바일 1열 명시) + **카드 클릭 시 라이트박스 팝업**.

추가 구현:
- `.gallery-grid` 반응형 열 수 명시(`@media`), `.gallery-item { cursor: zoom-in }`
- **라이트박스 CSS**(`.lightbox-overlay/.lightbox-content/.lightbox-img/.lightbox-caption/.lightbox-close`) — 어두운 오버레이 + `object-fit:contain` 전체 이미지 + 캡션 + 닫기(×)
- 라이트박스 HTML 컨테이너(`#gallery-lightbox`)
- **범용 JS**: `.gallery-item` 전체 이벤트 위임 → 항목 추가돼도 자동 팝업. 닫기: 배경 클릭 / × / ESC. 배경 스크롤 잠금. `data-full` 속성 지원(고해상도 팝업 선택).

검증: 자동클릭 스크린샷으로 팝업 동작 확인. 10개 항목 추가 대비 완료.
