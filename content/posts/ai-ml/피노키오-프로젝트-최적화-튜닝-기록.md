# 피노키오 프로젝트 — 최적화 튜닝 기록

> **날짜:** 2026-02-14
> **목표:** 사용자 발화 → AI 응답 체감 지연 최소화
> **결과:** 순차 처리 ~3.5초 → 병렬 처리 ~1.5초 (약 57% 단축)

---

## 1. 병목 분석

사용자 발화 후 AI 응답까지의 파이프라인과 각 구간 소요 시간:

```
사용자 발화 → [STT 대기] → [음성 분석] → [LLM 추론] → [TTS 합성] → AI 응답
              ~1.5초         1.18초        0.93초        ~0.1초
```

총 체감 지연: 약 3.5초 (STT pause_threshold 포함)

## 2. 최적화 항목

### 2.1 STT 파라미터 튜닝 (`config/settings.py`)

| 파라미터 | 변경 전 | 변경 후 | 효과 |
|----------|---------|---------|------|
| `energy_threshold` | 5000 | 15000 | 배경 잡음 필터링, 인식 정확도 향상 |
| `dynamic_energy` | True | False | 자동 임계값 상승 방지 (내장 DMIC 특성) |
| `phrase_time_limit` | 60초 | 15초 | 최대 녹음 시간 단축 |
| `pause_threshold` | (기본 0.8) | 1.5 → 1.0 | 발화 종료 판단 시간 단축 |
| `microphone_index` | None | 10 (PipeWire) | 내장 DMIC 직접 지정 |

**내장 마이크 진단 결과:**
- PipeWire가 DMIC(card 3: acp6x) 점유
- SpeechRecognition은 PipeWire 장치(index 10)로 접근
- 배경 RMS ~4248 vs 말소리 RMS ~17000 → 임계값 15000으로 고정

### 2.2 LLM 프롬프트 압축 (`core/llm_client.py`)

**변경 전 (~300자):**
```
[전략: 감정 탐색] 지금은 탐색 단계입니다. 사용자의 감정과 최근 경험에...
[감정: 불안 😰, 스트레스: 45, 레벨: medium]
[이상 감지: 시선 회피 빈번, 입술 압축 강함]
[영상 관찰: 시선 회피 많음, 미세표정(억제된 웃음)]
[음성 관찰: 응답 지연, 말 더듬]
[최근 대화]
  피노키오: 오~ 그렇군요!...
  사용자: 요즘 좀 힘들어요...
사용자: '직장에서 스트레스 받아요'
```

**변경 후 (~80자):**
```
[explore] 관찰:불안 😰,시선회피,억제된 웃음,응답지연 스트레스:45
이전답변:오~ 그렇군요! 그때 기분이 어땠어요?
사용자:직장에서 스트레스 받아요
```

**효과:** 입력 토큰 약 1/3로 감소 → EXAONE 추론 시간 단축

### 2.3 시스템 프롬프트 분리 (`core/llm_client.py`)

| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| 성격/말투 설명 | 매 호출 prompt에 포함 (~300자) | `system` 파라미터로 1회 전송 |
| prompt 내용 | 성격 + 상황 + 사용자 발화 | 데이터 + 사용자 발화만 |

### 2.4 LLM 생성 옵션 (`config/settings.py`)

| 파라미터 | 변경 전 | 변경 후 | 효과 |
|----------|---------|---------|------|
| `num_predict` | 무제한 | 80 | 최대 생성 토큰 제한 → 불필요한 토큰 생성 방지 |
| `temperature` | (기본 1.0) | 0.7 | 약간의 확정성 → 생성 속도 미세 향상 |
| `stream` | False | False (유지) | 추후 스트리밍 적용 시 추가 단축 가능 |

### 2.5 음성 분석 병렬화 (`main.py`)

**변경 전 (순차 처리):**
```
음성분석(1.18초) → LLM(0.93초) → TTS
총 대기: ~2.1초
```

**변경 후 (병렬 처리):**
```
음성분석(1.18초) ──┐
LLM(0.93초) ───────┤→ TTS
총 대기: ~1.0초 (max 기준)
```

구현: `threading.Thread`로 음성 분석 분리, `QTimer.singleShot(0, ...)` 로 UI 안전 갱신

### 2.6 프레임 루프 경량화 (`main.py`)

| 항목 | 변경 전 | 변경 후 | 효과 |
|------|---------|---------|------|
| `analyzer.analyze()` voice_data | 매 프레임 전달 | None (영상만) | 불필요한 음성 스코어 재계산 제거 |
| 음성 지표 UI | 매 프레임 체크 + "대기 중" 리셋 | 분석 완료 시만 갱신 | UI 갱신 부하 감소 |

## 3. 최적화 전후 비교

### 전체 파이프라인 시간

| 구간 | 최적화 전 | 최적화 후 |
|------|-----------|-----------|
| STT 대기 (pause) | ~1.5초 | ~1.0초 |
| 음성 분석 | 1.18초 (순차) | 1.18초 (병렬, 비대기) |
| LLM 추론 | ~2.5초 | ~0.93초 |
| TTS 합성 | ~0.1초 | ~0.1초 |
| **총 체감 지연** | **~3.5초** | **~1.5초** |

### LLM 추론 시간 상세

| 항목 | 최적화 전 | 최적화 후 |
|------|-----------|-----------|
| 프롬프트 크기 | ~300자 | ~80자 |
| 시스템 프롬프트 | prompt 내 반복 | system 분리 |
| 최대 토큰 | 무제한 | 80 |
| 측정 시간 | ~2.5초 | 0.93초 |

## 4. 추가 최적화 가능 항목 (미적용)

| 항목 | 예상 효과 | 비고 |
|------|-----------|------|
| LLM 스트리밍 (`stream: true`) | 첫 토큰 ~0.1초 만에 TTS 시작 가능 | TTS도 스트리밍 대응 필요 |
| EXAONE 2.4B 모델 | 추론 속도 2~3배 향상 | 응답 품질 하락 가능 |
| librosa → torchaudio 변환 | 피치 분석 0.5초 이내 | GPU 활용 가능 |
| Whisper 로컬 STT | Google API 네트워크 지연 제거 | GPU VRAM 추가 필요 |
| ONNX Runtime 추론 | MediaPipe 프레임 처리 최적화 | Phase 6 예정 |

## 5. 수정된 파일 목록

| 파일 | 변경 내용 |
|------|----------|
| `config/settings.py` | STT 파라미터 (에너지/pause/phrase), LLM 옵션 (num_predict/temperature) |
| `core/llm_client.py` | 시스템 프롬프트 분리, 프롬프트 압축, 대화 전략 |
| `utils/stt_engine.py` | 마이크 장치 지정, 에너지 임계값 수동 고정, pause_threshold |
| `main.py` | 음성 분석 병렬화, 프레임 루프 경량화, 음성 지표 UI 분리 |

## 6. 현재 설정값 요약

```python
# settings.py
STT = {
    "microphone_index": 10,
    "energy_threshold": 15000,
    "dynamic_energy": False,
    "pause_threshold": 1.0,
    "phrase_time_limit": 15,
    "timeout": 20,
}

LLM = {
    "model": "exaone3.5",       # 7.8B, 4.8GB
    "num_predict": 80,
    "temperature": 0.7,
    "stream": False,
    "timeout": 15,
}
```

> **최적화 결과: 체감 응답 지연 ~3.5초 → ~1.5초 (약 57% 단축). 프롬프트 압축 + 시스템 프롬프트 분리 + 음성 분석 병렬화가 핵심.**
