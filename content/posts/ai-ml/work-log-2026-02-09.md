### 19. 낙상 지속 알림 기능 구현 ⭐

**배경:** 현재 시스템은 "누워있는 자세"만 감지하며, 자발적 눕기(수면)와 의식 상실을 구분하지 못함. Fall 감지 후 일정 시간 부동 시 알림을 트리거하는 기능 필요.

**구현 방식:** Fall 감지 후 N초(기본 30초) 지속 시 비모달 팝업 알림

**수정 파일:** `monitoring_page.py` (1개 파일만 수정)

**변경 사항:**

| 위치 | 추가 내용 | 줄 수 |
|------|---------|------|
| `__init__` | 변수 4개: `fall_start_time`, `fall_alert_sent`, `fall_alert_threshold=30`, `emergency_dialog` | +5줄 |
| `process_stgcn_inference` Fall 블록 | 타이머 시작 → 30초 경과 시 로그 + DB(Emergency) + 비모달 팝업 | +11줄 |
| `process_stgcn_inference` Normal 블록 | 타이머 리셋 + 팝업 자동 닫기 | +5줄 |
| 클래스 하단 | `show_emergency_popup()`, `close_emergency_popup()` 메서드 | +40줄 |

**동작 흐름:**

```
Fall 감지 → 타이머 시작 → 30초 지속 → 로그 + DB(Emergency) + 비모달 팝업
                        → Normal 복귀 → 타이머 리셋 + 팝업 자동 닫기
```

**비모달 팝업 선택 이유:**
- `QMessageBox`(모달)는 확인 누를 때까지 추론이 멈춤
- `QDialog`(비모달)은 팝업 표시 중에도 추론 계속 동작
- Normal 복귀 시 팝업 자동 닫힘 → 오탐 시 자연스럽게 해소

**알림 시 기록:**
- 로그: `[EMERGENCY] ⚠️ 30초간 낙상 지속 — 의식 상실 의심!`
- DB: Type='Emergency'로 저장 (기존 Normal/Falling과 구분)

**테스트 결과:** ✅ 정상 동작 확인
