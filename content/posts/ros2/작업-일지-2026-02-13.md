### 2. ROS2 guard_brain 패키지 — LLM 노드 통합

**패키지 구조:**
```
home_guard_ws/src/guard_brain/
├── guard_brain/
│   ├── __init__.py
│   ├── brain_node.py          ← LLM 상황 판단 노드
│   └── sensor_sim_node.py     ← 센서 시뮬레이터 (테스트용)
├── setup.py
├── setup.cfg
├── package.xml
└── resource/guard_brain
```

**brain_node.py:**
- 구독: `/guard/sensor_data` (JSON 센서 데이터)
- 발행: `/guard/report` (JSON 구조화 보고서)
- 발행: `/guard/tts_command` (TTS 대사 문자열)
- Ollama EXAONE 3.5 7.8B 호출 → JSON 파싱 → 토픽 발행

**sensor_sim_node.py:**
- 15초 간격으로 4개 시나리오 순환 발행
- 시나리오: 정상 순찰, 낙상 감지, 가스 누출, 빈 공간
- 센서값에 랜덤 변동 추가 (온도 ±1°C, 습도 ±3%, 배터리 감소)

### 3. ROS2 + Ollama 연동 이슈 해결

**문제:** `ros2 run guard_brain brain_node` 실행 시 `ModuleNotFoundError: No module named 'ollama'`

**원인:** ROS2 엔트리포인트 shebang이 `#!/usr/bin/python3` (시스템 Python)을 가리켜서 `ros2_venv`의 패키지를 참조하지 못함

**해결:** `python -m guard_brain.brain_node` 방식으로 직접 실행
- `ros2_venv`에 누락 패키지 추가 설치: `ollama`, `PyYAML`, `numpy`

### 4. 통합 테스트 결과

**3개 터미널 동시 실행:**
- 터미널 1: `python -m guard_brain.brain_node` (LLM 판단)
- 터미널 2: `python -m guard_brain.sensor_sim_node` (센서 시뮬레이션)
- 터미널 3: `ros2 topic echo /guard/tts_command` (모니터링)

**테스트 결과 (4개 시나리오 모두 성공):**

| 시나리오 | 등급 | TTS 대사 | 추론 시간 |
|---------|------|---------|----------|
| 거실 정상 | [정상] | "정상 상태입니다. 계속 감시하겠습니다." | ~14초 |
| 침실 낙상 | [위험] | "위험! 낙상 감지되었습니다. 즉시 확인하고 연락하세요." | ~17초 |
| 주방 가스 | [경고] | "경고! 주방 가스 누출 위험, 가스 차단하세요." | ~16초 |
| 베란다 무인 | [주의] | "주의! 베란다 습도가 높고 배터리 충전 확인 필요합니다." | ~14초 |
