## 2. ROS2 엔트리포인트 Shebang 근본 해결 ✅

### 문제
`colcon build` 시 엔트리포인트 shebang이 `#!/usr/bin/python3` (시스템 Python)으로 강제 고정되어, `ros2_venv`의 패키지(ollama 등)를 `ros2 run`에서 import 할 수 없었다.

### 원인 분석
- `ros2_venv` activate 상태에서 `which python3`는 venv Python을 가리킴
- 그럼에도 `colcon build`(setuptools)가 shebang을 시스템 Python으로 재작성
- ROS2 Jazzy + ament_python 빌드 타입의 알려진 동작

### 해결 (하이브리드 방식)
`build.sh`에서 `colcon build` 완료 후, `install/*/lib/*/*` 경로의 모든 엔트리포인트를 스캔하여 shebang을 venv Python 경로로 자동 패치한다.

```bash
# 패치 핵심 로직
sed -i "1s|.*|#!/home/gjkong/dev_ws/ros2_venv/bin/python3|" install/guard_brain/lib/guard_brain/brain_node
```

### 산출물

| 파일 | 경로 | 역할 |
|------|------|------|
| build.sh | `scripts/build.sh` | venv activate → colcon build → shebang 자동 패치 |
| run.sh | `scripts/run.sh` | venv + ROS2 + install source → ros2 run 자동화 |
| guard_brain.launch.py | `home_guard_ws/src/guard_brain/launch/` | 표준 Node() 방식 launch |

### 테스트 결과
```
./scripts/run.sh brain_node
🤖 실행: ros2 run guard_brain brain_node
   Python: /home/gjkong/dev_ws/ros2_venv/bin/python3
[INFO] 🤖 Guard Brain 시작 — 모델: exaone3.5:7.8b    ✅
```

## 3. guard_voice 노드 구현 ✅

### 개요
`/guard/tts_command` 및 `/guard/report` 토픽을 구독하여, Edge TTS(Microsoft)로 한국어 여성 음성을 합성하고 sounddevice로 스피커 재생하는 ROS2 노드를 구현하였다.

### 기술 스택

| 항목 | 선택 | 비고 |
|------|------|------|
| TTS 엔진 | Edge TTS (Microsoft) | 온라인, 고품질 한국어 여성 음성 |
| 음성 | ko-KR-SunHiNeural (기본) | ko-KR-JiMinNeural 대안 |
| 오디오 재생 | sounddevice | 비동기, 중단 지원 |
| MP3→WAV 변환 | pydub + ffmpeg | — |

### 재생 정책: 우선순위 큐 하이브리드

| 등급 | 우선순위 | 동작 |
|------|---------|------|
| 위험 | 0 (최고) | 현재 재생 즉시 중단 → 긴급 재생 |
| 경고 | 1 | 현재 재생 즉시 중단 → 긴급 재생 |
| 주의 | 2 | 큐 추가 → 순서대로 재생 |
| 정상 | 3 | 큐 추가 → 순서대로 재생 |

### ROS2 인터페이스

| 토픽 | 방향 | 메시지 | 설명 |
|------|------|--------|------|
| `/guard/tts_command` | 구독 | String | 단순 텍스트 대사 |
| `/guard/report` | 구독 | String (JSON) | level + tts_voice 포함 보고서 |

### 파라미터

| 파라미터 | 기본값 | 설명 |
|---------|--------|------|
| voice | ko-KR-SunHiNeural | Edge TTS 음성 |
| volume | 1.0 | 볼륨 (0.0~1.0) |
| rate | +0% | 속도 조절 |
| pitch | +0Hz | 피치 조절 |
| cache | true | 동일 대사 캐시 |

### 패키지 구조
```
home_guard_ws/src/guard_voice/
├── guard_voice/
│   ├── __init__.py
│   └── tts_node.py
├── resource/
│   └── guard_voice
├── package.xml
├── setup.cfg
└── setup.py
```

### 추가 설치 패키지 (ros2_venv)
```bash
pip install edge-tts sounddevice pydub soundfile
sudo apt install -y ffmpeg
```

### 통합 테스트 결과
```
sensor_sim → /guard/sensor_data → brain_node → LLM 추론
    → /guard/report + /guard/tts_command → tts_node → 🔊 스피커 재생    ✅
```

3개 노드(brain_node + sensor_sim_node + tts_node) 동시 실행 시 전체 파이프라인 정상 동작 확인.
