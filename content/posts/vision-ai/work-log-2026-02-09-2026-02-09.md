### 19. 낙상 지속 알림 기능 구현 ⭐

**배경:** 현재 시스템은 자세만 감지하여 자발적 눕기(수면)와 의식 상실을 구분하지 못함

**구현:** Fall 감지 후 30초 지속 시 비모달 팝업 알림

**수정 파일:** `monitoring_page.py` (1개 파일, 기존 코드 수정 없이 추가만)

| 위치 | 추가 내용 | 줄 수 |
|------|---------|------|
| `__init__` | `fall_start_time`, `fall_alert_sent`, `fall_alert_threshold=30`, `emergency_dialog` | +5줄 |
| Fall 블록 | 타이머 시작 → 30초 경과 시 로그 + DB(Emergency) + 비모달 팝업 | +11줄 |
| Normal 블록 | 타이머 리셋 + 팝업 자동 닫기 | +5줄 |
| 클래스 하단 | `show_emergency_popup()`, `close_emergency_popup()` | +40줄 |

**동작 흐름:**

```
Fall 감지 → 타이머 시작 → 30초 지속 → 로그 + DB(Emergency) + 비모달 팝업
                        → Normal 복귀 → 타이머 리셋 + 팝업 자동 닫기
```

**비모달 선택 이유:** QMessageBox(모달)는 확인 누를 때까지 추론이 멈추므로, QDialog(비모달)로 구현하여 추론 계속 + Normal 복귀 시 자동 닫힘

**테스트 결과:** ✅ 정상 동작

### 20. 비전 AI 숨은그림찾기 게임 개발 ⭐⭐

**프로젝트:** `gaze_puzzle/` — 검지 포인팅으로 사진 속 코믹 낙서를 찾는 게임

#### 개발 과정

**1단계: 시선 추적 실험 (폐기)**

- MediaPipe Face Mesh 468 랜드마크 + 홍채(468~477) 추적 시도
- 자동 시선 테스트 시퀀스로 정밀도 측정:
  - x 범위: 0.538~0.552 (차이 0.014) — 거의 변화 없음
  - y 범위: 0.383~0.454 (차이 0.071) — 약간 변화
- **결론:** 일반 웹캠으로는 게임에 쓸 수 있는 시선 정밀도 확보 불가
- NumPy 2.x ↔ MediaPipe 버전 충돌 → 별도 venv (`gaze_venv`) 구축

**2단계: 검지 포인팅으로 전환 (채택)**

- MediaPipe Hands 21개 손 관절 추적
- INDEX_FINGER_TIP (keypoint 8) 좌표 → 화면 커서
- 포인팅 제스처 판별: 검지만 펴면 초록, 아니면 주황
- 포인팅 중일 때만 HIT 판정
- `cv2.flip` 거울 모드 이중 반전 버그 수정

**3단계: 퍼즐 생성기**

- 초기: 도형 기반 퍼즐 (6종 도형, 5종 변형)
- 개선: 사진 기반 + 코믹 낙서 방식으로 전환
- 블러 기반 변형 → 너무 눈에 띄어서 폐기
- **최종:** 코믹 낙서 14종을 사진 위에 추가

**4단계: 전체 통합**

- 1280x720 전체화면 게임
- 웹캠 PIP (우하단 200x150)
- Dwell 판정 0.8초 + 판정 범위 축소 (radius - 5)
- 대기 화면 슬라이드쇼 (2.5초 간격, 랜덤 시작)
- 레벨 시스템 + 점수 시스템

## 📌 기술 교훈

1. **시선 추적 한계:** 일반 웹캠 + MediaPipe Face Mesh로는 정밀한 시선 추적 불가 (홍채 움직임 범위가 눈 대비 2~7%에 불과)
2. **NumPy 버전 충돌:** MediaPipe + TensorFlow는 numpy<2 필요, YOLO/PyTorch는 numpy 2.x 호환 → 별도 venv 필수
3. **거울 모드 주의:** `cv2.flip` + 좌표 반전 시 이중 반전 버그 주의
4. **블러 vs 코믹:** 사진 변형 시 블러 블렌딩은 너무 눈에 띄어서 게임으로 부적합, 코믹 낙서 방식이 게임성에 적합
5. **상대 경로:** `os.path.dirname(os.path.abspath(__file__))` 사용으로 어디서든 실행 가능
