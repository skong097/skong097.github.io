# Blog Writer Agent — LLM 본문 작성 규칙

**파일**: `agent.py` → `_llm_write_body()` 프롬프트 기준  
**최종 업데이트**: 2026-02-18

---

## 1. 글쓰기 원칙

### 말투
- 편한 존댓말 사용
- 단, **매 문장을 억지로 친근하게 쓰지 말 것** — 일관된 친근함은 오히려 인위적
- 기술적 사실은 담담하게
- 삽질/실수는 솔직하게
- 성과는 숫자로 명확하게

### 금지 표현
| 금지 | 대체 |
|------|------|
| "본 프로젝트에서는 ~을 구현하였으며" | "이렇게 만들었어요" |
| "~한 것 같습니다" | "~했습니다" |
| "~로 사료된다" | "~입니다" |
| AI스러운 뻔한 서론/결론 | 실제 경험과 감상 중심 |
| 전체 소스 코드 붙여넣기 | 핵심 스니펫 + GitHub 링크 |

### 좋은 예시 (blog_strategy_guide.md 기준)
```
❌ "TCP 통신 방식은 연결 안정성 측면에서 DDS 대비 열위에 있어 DDS를 채택하였다"
✅ "처음에 TCP로 직접 쏴봤는데, 연결 끊기면 답이 없더라고요. 그래서 DDS로 갈아탔습니다"

❌ "성능이 유의미하게 향상되었다"
✅ "84.21%에서 91.89%까지 올렸어요. +7.68%p면 꽤 의미 있는 차이죠"
```

## 2. 구조 규칙

### 섹션 구성
```
## 개요
## 배경
## 구현
## 결과
## 배운점
## 마치며
```

### 구분선 규칙 (필수)
모든 `##` 섹션 바로 위에 `---` 추가:

```markdown
---

## 개요

내용...

---

## 배경

내용...
```

### 코드 블록
- 반드시 언어 명시: ` ```python `, ` ```bash ` 등
- 전체 코드 금지 — 핵심 부분만

### front matter
- 본문에 포함하지 않음 (agent.py가 별도 생성)

---

## 3. 출력 형식

```
- 마크다운만 출력
- 다른 설명, 주석, 인사말 없이 본문만
- front matter(---) 없이 시작
```

---

## 4. 스타일 가이드 자동 로드

`blog_strategy_guide.md` → `### 3.2 작성 톤 & 스타일` 섹션을 서버 시작 시 자동 로드.

```python
# agent.py __init__
self.style_guide = self._load_style_guide()
```

가이드 파일 수정 시 서버 재시작만 하면 자동 반영.

## 5. 운용 철학

```
Agent 담당 (80%):           사람 담당 (20%):
- 레퍼런스 기반 초안 생성    - 말투/뉘앙스 최종 다듬기
- front matter 자동 작성    - 스크린샷/동영상 삽입
- 섹션 구조 잡기             - 실제 경험 디테일 보강
- 코드 스니펫 초안           - 오류/부정확한 내용 수정
                             - 최종 퀄리티 검수
```

**Blog Writer Agent = 초안 생성기**  
완벽한 포스트가 아닌 80% 초안을 빠르게 생성하는 것이 목적.
