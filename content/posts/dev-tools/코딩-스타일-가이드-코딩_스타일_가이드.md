## 2. C++ 코딩 스타일 가이드

> 기준: [Google C++ Style Guide](https://google.github.io/styleguide/cppguide.html)

### 2.1 네이밍 규칙

| 구분 | 규칙 | 예시 |
|------|------|------|
| 파일명 | snake_case, 소문자 | `my_class.cpp`, `utils.h` |
| 클래스 / 구조체 | PascalCase | `MyClass`, `DataPacket` |
| 함수 / 메서드 | PascalCase | `GetValue()`, `SendData()` |
| 변수 (지역 / 파라미터) | snake_case | `read_value`, `buffer_size` |
| 멤버 변수 | snake_case + 후위 언더스코어 | `device_id_`, `data_buffer_` |
| 전역 변수 | `g_` 접두사 + snake_case | `g_device_state` |
| 상수 (constexpr / const) | kPascalCase | `kMaxRetry`, `kBufferSize` |
| 매크로 | ALL_CAPS (가능하면 constexpr 사용) | `MAX_BUFFER_SIZE` |
| 네임스페이스 | 소문자 snake_case | `iot::network`, `iot::sensor` |
| 열거형 (enum class) | PascalCase (값은 kPascalCase) | `enum class DeviceState { kIdle, kActive };` |

> ⚠️ `using namespace std;` 는 헤더 파일에서 절대 사용하지 않습니다. 구현 파일에서도 최소화합니다.

### 2.4 헤더 파일

- `#pragma once` 를 헤더 가드로 사용합니다.
- Include 순서: **관련 헤더 → C 표준 → C++ 표준 → 서드파티 → 프로젝트 내부**. 각 그룹 사이 빈 줄을 둡니다.
- 헤더에는 최소한의 선언만 포함하고, 구현은 `.cpp` 에 작성합니다.

```cpp
#pragma once

#include <cstdint>
#include <string>
#include <memory>

#include "third_party/some_lib.h"

#include "project/internal_module.h"
```

### 2.5 메모리 & 포인터

| 규칙 | 설명 |
|------|------|
| 원시 포인터 최소화 | 소유권이 필요한 경우 스마트 포인터를 사용합니다. |

### 2.6 클래스 설계

- 접근 지정자 순서: `public` → `protected` → `private`
- 각 접근 지정자 안에서: 타입 → 상수 → 생성자/소멸자 → 함수 → 멤버 변수 순으로 작성합니다.
- 복사 생성자 / 복사 대입 연산자가 필요 없는 경우 `= delete` 로 명시합니다.
- 상속 시 소멸자는 반드시 `virtual` 로 선언합니다.

### 2.7 문자열 포매팅 (C++20)

C++20 이상을 사용하는 경우 문자열 포매팅은 **`std::format`** 을 우선합니다. C++20 미만 환경에서는 `{fmt}` 라이브러리를 사용하며, `printf` 및 `std::stringstream` 사용을 지양합니다.

| 방식 | 사용 기준 |
|------|-----------|
| `std::format` | **C++20 이상 — 기본 선택** |
| `fmt::format` | C++20 미만 환경 또는 `{fmt}` 라이브러리 사용 시 |
| `std::stringstream` | 지양 (성능 및 가독성 열세) |
| `printf` / `sprintf` | 지양 (타입 안전성 없음) |

```cpp
// ✅ Good (C++20)
#include <format>

std::string msg = std::format("robot_{}: battery={:.1f}%", robot_id, battery);
RCLCPP_INFO(get_logger(), "%s", msg.c_str());

// ✅ Good (C++17 이하, {fmt} 라이브러리)
#include <fmt/format.h>

std::string msg = fmt::format("robot_{}: battery={:.1f}%", robot_id, battery);

// ❌ Bad
char buf[128];
sprintf(buf, "robot_%d: battery=%.1f%%", robot_id, battery);

std::stringstream ss;
ss << "robot_" << robot_id << ": battery=" << battery << "%";
```

## 3. Python 코딩 스타일 가이드

> 기준: [PEP 8 – Style Guide for Python Code](https://peps.python.org/pep-0008/)

### 3.1 네이밍 규칙

| 구분 | 규칙 | 예시 |
|------|------|------|
| 모듈 / 패키지 | 소문자 snake_case | `data_handler.py`, `utils/` |
| 클래스 | PascalCase | `DataHandler`, `DeviceManager` |
| 함수 / 메서드 | snake_case | `send_data()`, `read_value()` |
| 변수 | snake_case | `read_value`, `retry_count` |
| 상수 | ALL_CAPS | `MAX_RETRY = 3`, `TIMEOUT_SEC = 5` |
| 내부용 속성/메서드 | `_single_underscore` | `_internal_state`, `_connect()` |
| Name mangling (상속 충돌 방지용) | `__double_underscore` | `__private_buffer` |
| 매직 메서드 | `__dunder__` | `__init__`, `__repr__`, `__str__` |

> ⚠️ `__double_underscore`(더블 언더스코어)는 Python name mangling을 유발합니다.
> 단순 private 표현이 목적이라면 `_single_underscore`를 사용하세요.
> `__` 는 상속 계층에서 속성 이름 충돌을 방지해야 하는 경우로 한정합니다.

### 3.3 타입 힌트 (Type Hint)

Python 3.10+ 기준으로 타입 힌트를 **필수** 적용합니다.

| 항목 | 규칙 |
|------|------|
| 함수 파라미터 | 모든 파라미터에 타입 힌트 명시 |
| 반환 타입 | `->` 반환 타입 필수 명시 |
| 컨테이너 타입 | `list[int]`, `dict[str, int]` (소문자, Python 3.10+) |
| 선택적 타입 | `Optional[str]` 또는 `str \| None` (Python 3.10+) |
| 클래스 속성 | `self._r: Optional[redis.Redis] = None` 형태로 선언 |
| 변경 불가 상수 | `Final` 로 명시 — 재할당 방지 및 의도 명확화 |
| 허용 값 제한 | `Literal` 로 명시 — 특정 값만 허용하는 파라미터에 적용 |

```python
# ✅ Good — 기본 타입 힌트
def get_robot_state(self, robot_id: int) -> Optional[RobotState]:
    ...

def set(self, key: str, value: str, ttl_sec: int = -1) -> bool:
    ...

def get_alive_robots(self) -> list[int]:
    ...

# ✅ Good — Final: 모듈 레벨 상수에 재할당 방지
from typing import Final

MAX_RETRY: Final = 3
TIMEOUT_SEC: Final[float] = 5.0
ROBOT_COUNT: Final[int] = 5

# ✅ Good — Literal: 허용 값을 타입으로 제한
from typing import Literal

def set_mode(mode: Literal["auto", "manual"]) -> None:
    ...

def get_qos(policy: Literal["BEST_EFFORT", "RELIABLE"]) -> QoSProfile:
    ...

# ❌ Bad — 타입 힌트 없음
def get_robot_state(self, robot_id):
    ...

# ❌ Bad — Final 없이 상수 선언 (재할당 위험)
MAX_RETRY = 3
```

### 3.4 주석 & 독스트링

**인라인 주석**은 코드와 같은 줄에 최소 2칸 띄운 후 `#` 으로 작성합니다. 자명한 내용의 주석은 생략합니다.

**독스트링(Docstring)** 은 함수, 클래스, 모듈에 반드시 작성하며 **Google 스타일**을 사용합니다.

```python
def some_function(param_a: int, param_b: str = "default") -> bool:
    """함수의 역할을 한 줄로 요약합니다.

    Args:
        param_a: 첫 번째 파라미터 설명.
        param_b: 두 번째 파라미터 설명. 기본값은 "default".

    Returns:
        처리 성공 여부를 bool로 반환합니다.

    Raises:
        ValueError: param_a가 음수인 경우.
    """
    ...
```

### 3.5 문자열 포매팅

문자열 포매팅은 **`f-string`** 을 원칙으로 합니다. `.format()` 및 `%` 방식은 사용하지 않습니다.

| 방식 | 사용 기준 |
|------|-----------|
| `f"..."` | **기본 선택** — 가독성과 성능 모두 우수 |
| `str.format()` | 지양 (f-string으로 대체) |
| `%` 포매팅 | 금지 |
| `+` 문자열 연결 | 단순 결합 외 금지 (루프 내 특히 금지) |

```python
# ✅ Good
robot_id = 2
battery  = 85.3
msg = f"[robot_{robot_id}] battery={battery:.1f}%"
logger.info(f"[Blackboard] Redis 연결 실패: {e}")

# ❌ Bad
msg = "[robot_%d] battery=%.1f%%" % (robot_id, battery)
msg = "[robot_{}] battery={:.1f}%".format(robot_id, battery)
msg = "[robot_" + str(robot_id) + "] battery=" + str(battery) + "%"
```

### 3.7 예외 처리

재난 환경 로봇 코드의 특성상 예외 처리 일관성이 특히 중요합니다.

| 규칙 | 설명 |
|------|------|
| `bare except` 금지 | `except:` 단독 사용 금지 |
| 구체적 예외 타입 명시 | `except RedisError as e:`, `except ValueError as e:` |
| `print` 금지 | 노드 코드에서 `print()` 대신 `logging` 또는 `rclpy` 로거 사용 |
| 예외 로깅 형식 | `logger.error(f"[모듈명] 상황 설명: {e}")` |
| 예외 무시 금지 | 불가피한 경우 `# noqa` + 주석으로 사유 명시 |

```python
# ✅ Good
try:
    result = self._r.ping()
except RedisConnectionError as e:
    logger.error(f"[Blackboard] Redis 연결 실패: {e}")
    return None

# ❌ Bad
try:
    result = self._r.ping()
except:
    pass
```

### 4.4 실시간성 & 동적 할당 제한 (C++ / ROS2)

재난 환경 로봇 시스템에서 제어 루프와 고주파 콜백의 실시간성은 안정성에 직결됩니다.

#### 동적 메모리 할당 제한

| 규칙 | 설명 |
|------|------|
| 제어 루프 내 동적 할당 금지 | `new`, `std::make_shared`, `std::vector::push_back` 등 지양 |
| 사전 할당 원칙 | 노드 초기화(`init`) 단계에서 버퍼 사전 할당 후 재사용 |
| 고주파 콜백 내 임시 객체 금지 | `POSE_QOS`(10~20Hz) 등 고주파 콜백에서 임시 `std::string`, `std::vector` 생성 금지 |

```cpp
// ✅ Good — 초기화 단계에서 사전 할당
class PoseCallback : public rclpy::Node {
public:
  PoseCallback() {
    pose_buffer_.reserve(5);  // 초기화 시 사전 할당
  }

private:
  void OnPose(const PoseMsg& msg) {
    pose_buffer_[msg.robot_id] = msg.pose;  // 재사용
  }
  std::vector<Pose> pose_buffer_;
};

// ❌ Bad — 콜백마다 동적 할당 발생
void OnPose(const PoseMsg& msg) {
  std::vector<Pose> temp;          // 매 호출마다 할당
  temp.push_back(msg.pose);
}
```

#### 로그 폭주 방지

고주파 콜백 내 일반 로그(`RCLCPP_INFO`)는 초당 수백 건의 로그를 발생시켜 성능을 저하시킵니다. **Throttle 매크로**를 사용하세요.

```cpp
// ✅ Good — 1000ms(1초)마다 1회만 출력
RCLCPP_INFO_THROTTLE(get_logger(), *get_clock(), 1000,
    "robot_%d pose: (%.2f, %.2f)", robot_id, x, y);

RCLCPP_WARN_THROTTLE(get_logger(), *get_clock(), 5000,
    "battery low: %.1f%%", battery);

// ❌ Bad — 20Hz 콜백에서 매 호출 출력 (초당 20회)
RCLCPP_INFO(get_logger(), "robot_%d pose: (%.2f, %.2f)", robot_id, x, y);
```

| 타입 | 설명 | 예시 |
|------|------|------|
| `feat` | 새 기능 추가 | `feat(blackboard): add redis connection retry logic` |
| `fix` | 버그 수정 | `fix(slam): resolve map merge deadlock on robot_3` |
| `docs` | 문서만 변경 | `docs(guide): add type hint section` |
| `refactor` | 기능 변경 없는 코드 리팩터링 | `refactor(swarm): extract election guard to separate class` |
| `test` | 테스트 추가 / 수정 | `test(m03): add semantic memory blocked zone test` |
| `chore` | 빌드, 설정, 패키지 관리 | `chore(deps): update redis to 7.3.0` |
| `perf` | 성능 개선 | `perf(pose_callback): pre-allocate pose buffer` |
| `style` | 코드 스타일 변경 (로직 무관) | `style(blackboard): apply f-string formatting` |

#### 규칙

- `subject`는 명령형 현재 시제로 작성합니다. (`add`, `fix`, `update` — `added`, `fixed` 금지)
- `subject`는 72자 이내로 작성합니다.
- `scope`는 변경된 패키지 또는 모듈명을 사용합니다. (`blackboard`, `slam`, `swarm`, `victim` 등)
- 브레이킹 체인지는 footer에 `BREAKING CHANGE:` 로 명시합니다.

```bash
# ✅ Good
feat(blackboard): add redis sentinel HA support
fix(frontier): fix atomic claim race condition under high load
docs(m02): update interface file placement guide
test(m01): add pyjson5 fallback parser for zenoh config

# ❌ Bad
fixed bug
update code
작업 완료
```
