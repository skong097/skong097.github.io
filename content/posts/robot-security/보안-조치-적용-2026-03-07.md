---

## 완료 항목

### ✅ HIGH-1 — FastAPI JWT 인증 엔드포인트 전체 적용

- **파일:** `server/api_routes.py`
- **내용:** 기존 7개 + 신규 14개 = 총 21개 엔드포인트에 `Depends(verify_token)` 적용
- **신규 적용 엔드포인트:**
  - STT: `/stt/activate`, `/stt/mic-device`, `/stt/transcribe-audio`
  - SmartGate: `/smartgate/reload-faces`, `/smartgate/status`, `/smartgate/registered-faces`
  - Logs: `/logs/categories`, `/logs/stats`, `/logs/pattern/*`, `/logs/{log_id}`
- **검증:** 토큰 없이 → 401, 토큰 포함 → 200 ✅

---

### ✅ HIGH-2 — TCP HMAC 서명 적용

- **파일:** `server/command_router.py`, `server/esp32_secure.py`
- **내용:**
  - `_sign_payload()` 헬퍼 함수 추가
  - ESP32로 나가는 TCP 전송 10개 전체에 HMAC 서명 적용
  - `.env`에 `ESP32_SECRET` 추가
- **서명 포맷:** `{"ts": "...", "sig": "...", "cmd": ...}`
- **비고:** ESP32 펌웨어 측 검증은 esp32_home 연결 시 별도 진행 예정 (`docs/esp32_hmac_verify.cpp`)

---

### ✅ HIGH-3 — WebSocket JWT 인증 (403 디버깅 완료)

- **1차 원인:** 서버 구버전 실행 중 → `run_server.sh` 재시작으로 해결
- **2차 원인:** `run_server.sh`에서 매 실행 시 `JWT_SECRET` 재생성 → 브라우저 토큰 무효화
- **수정 내용:** `run_server.sh` JWT_SECRET 조건 수정 — 값이 없을 때만 최초 1회 생성

```bash
# 기존 — 매번 재생성
if ! grep -q "JWT_SECRET" .env || grep -q "JWT_SECRET=$" .env

# 변경 — 최초 1회만 생성
if ! grep -q "^JWT_SECRET=.+" .env
```

- **3차 원인:** 브라우저 탭 중복 (이전 탭이 만료 토큰으로 재연결 시도) → 탭 정리 후 해결 ✅

---

### ✅ MEDIUM-6 — settings.yaml 권한 제한 및 민감값 분리

- **파일:** `config/settings.yaml`, `server/main.py`, `.env`
- **민감값 분리 내용:**

| 항목 | 기존 위치 | 변경 후 |
|------|-----------|---------|
| `database.user` | settings.yaml 평문 | `.env DB_USER` |
| `database.password` | settings.yaml 평문 | `.env DB_PASSWORD` |
| `smartgate.gesture_auth.sequence` | settings.yaml 평문 | `.env SMARTGATE_SEQUENCE` |

- **main.py 추가:** `SMARTGATE_SEQUENCE` 오버라이드 파싱 로직
- **파일 권한:**
  ```bash
  chmod 600 config/settings.yaml
  chmod 600 .env
  chmod 700 face_db/
  ```
- **검증:** `DBLogger 생성: localhost:3306/iot_smart_home` ✅
