## STEP 3 — FastAPI JWT 인증 적용 ✅

**보안 기준:** NIST SP 800-213 §4.3 / OWASP OT2

### 해결한 이슈: WebSocket 403 반복

**증상:** 브라우저 대시보드 WebSocket `/ws` 연결 시 지속적으로 403 거부

**원인 분석 과정:**
1. `sessionStorage.getItem('jwt_token')` → 토큰 정상 존재 확인
2. 토큰 만료 시간 확인 → 유효 (만료까지 40분 이상)
3. 직접 WS 연결 테스트 → `ws://localhost:8000/ws?token=xxx` 성공
4. 서버 재시작 후 로그 확인 → `[accepted]` 정상 수락 확인

**근본 원인:** 서버가 이전 버전(JWT 적용 전)으로 실행 중이었음  
**해결:** `./run_server.sh` 재시작으로 완전 해결

### JWT 보호 엔드포인트 신규 적용 (21개 전체)

## STEP 4 — TCP :9000 HMAC 서명 적용 ✅

**보안 기준:** NIST SP 800-213 §4.2 / OWASP OT1, OT3

### 구현 방식

`esp32_secure.py`의 `build_signed_packet()` 을 `command_router.py`에 연동.  
서버 → ESP32 모든 TCP 전송에 HMAC-SHA256 서명 패킷 적용.

```python
# command_router.py 상단에 추가된 헬퍼
try:
    from server.esp32_secure import build_signed_packet as _build_signed_packet
    _HMAC_ENABLED = True
except ImportError:
    _HMAC_ENABLED = False  # 폴백: 평문 전송

def _sign_payload(payload: bytes) -> bytes:
    """평문 payload → HMAC 서명 패킷 (실패 시 원본 반환)"""
    if not _HMAC_ENABLED:
        return payload
    try:
        cmd_dict = json.loads(payload.decode().strip())
        signed = _build_signed_packet(cmd_dict)
        return (signed + "\n").encode()
    except Exception:
        return payload  # 폴백
```

### 적용된 TCP 전송 (10개 전체)

```
send_command(device_id,      _sign_payload(payload))     ← LED/Servo 단일 명령
send_command(led_device_id,  _sign_payload(payload))     ← all_off/all_on LED
send_command(target,         _sign_payload(payload))     ← Servo 전체
send_command(pir_device,     _sign_payload(payload))     ← PIR 모드 (guard/presence/dnd)
send_command(pir_device,     _sign_payload(payload_led)) ← LED 복원 (home/wake mode)
send_command(seg7_device,    _sign_payload(message))     ← 욕실 온도 7세그
```

### ESP32_SECRET 설정

```bash
# .env에 자동 생성 완료
ESP32_SECRET=4bf34c04e9cff8985d2f4f156de874da
```

### 서명 패킷 검증

```python
# 서명 정상 출력 확인
result = build_signed_packet({'cmd':'led','state':'on'})
# → b'{"ts": "1772858717", "sig": "91ade2b4e7a6...", "cmd": ...}'  ✅
```
