## 배경

MOCA는 카페에서 손님에게 모객·서빙·안내를 자율로 수행하는 모바일 로봇입니다. 모객 단계에서는 손님의 표정을 실시간으로 살펴, **반응이 좋으면 다가가고, 불편해 보이면 멈춰서 물러나는** 동작이 필요했습니다.

감정 신호 자체는 표정 인식 파이프라인이 Valence-Arousal 값으로 흘려보내 줍니다. 문제는 **이 신호를 어떻게 행동에 반영할 것인가**였습니다.

- 매 순간의 표정을 그대로 반영하면, 잠시 인상을 찌푸리거나 햇빛에 눈을 가린 손님에게도 모객을 중단해 버립니다.
- 그렇다고 신호를 너무 부드럽게 평활하면, 손님이 명백히 불쾌해하는 순간에도 로봇이 계속 다가가는 사고가 납니다.

즉 **과민반응과 둔감 사이의 균형**이 핵심이었습니다.

## 접근

### 1) 트리 최상위 — ReactiveFallback

MOCA의 모객 시나리오는 6단계로 구성된 BehaviorTree로 동작합니다. 그 최상위를 일반적인 Sequence/Fallback이 아닌, **`ReactiveFallback`**으로 두었습니다.

`ReactiveFallback`은 자식 노드를 매 tick마다 위에서부터 다시 평가합니다. 따라서 다음 두 가지 가드를 트리의 가장 위에 올려두면, 모객의 어느 단계가 진행 중이든 매 tick 안전·감정 상태가 재평가됩니다.

```
ReactiveFallback
├── 안전 가드 (장애물/사람 충돌 위험 → 즉시 정지)
├── 감정 가드 (지속 부정 감정 → 모객 중단/물러나기)
└── 모객 시퀀스 (IDLE → APPROACH → ICEBREAK → MINIGAME → OFFER → LEAD-IN)
```

이 구조에서는 **부정 감정이 감지되는 순간 모객 시퀀스가 즉시 우선순위에서 밀려납니다.** Reactive 키워드가 그것을 보장합니다.

### 2) 감정 신호 자체는 부드럽게 — 신뢰도 가중 EMA

가드가 매 tick 반응한다고 해서 **노이즈 한 프레임에도 멈추면** 곤란합니다. 카메라가 잠깐 흔들리거나 손이 얼굴을 가린 순간에도 모객이 중단되어 버립니다.

그래서 감정 신호를 트리에 넣기 전에 한 단계 평활하였습니다.

- 표정 인식 결과에는 **인식 신뢰도(confidence)**가 같이 따라옵니다. 측면을 향했거나 부분 가림이 있을 때 낮게 떨어집니다.
- 단순 평균이 아닌 **신뢰도 가중 EMA(Exponential Moving Average)**로 valence·arousal을 갱신합니다. 신뢰도가 낮은 프레임은 과거 값에 거의 영향을 주지 못하도록 가중치가 낮게 들어갑니다.

```
ema_t = α · w_t · x_t + (1 − α · w_t) · ema_{t−1}
  x_t    : 현재 감정 값
  w_t    : 인식 신뢰도 (0~1)
  α      : 기본 평활 강도
```

이렇게 하면 신뢰도가 떨어지는 노이즈 프레임은 평균을 거의 흔들지 않고, 신뢰도가 높은 안정적인 프레임만 평균을 끌어당깁니다.

### 3) 가드 발동 조건 — 히스테리시스

가드가 EMA 값 하나로 매 tick 판단하면, 임계값 근처에서 ON/OFF가 떨리는 현상(채터링)이 생길 수 있습니다. 모객을 멈췄다가 재개했다가 다시 멈추는 동작은 손님에게 더 큰 불쾌감을 줍니다.

그래서 임계값을 **두 개**로 둡니다(히스테리시스).

- **진입 임계** — EMA valence가 이 값보다 낮게 **N 프레임 연속** 유지되면 가드 발동.
- **해제 임계** — 가드 발동 후, EMA valence가 이 값보다 **여유 있게 회복**되어야만 가드 해제.

진입 임계가 해제 임계보다 낮은 곳에 위치하기 때문에, 임계 근처에서 한 번 들어간 가드는 쉽게 풀리지 않고, 풀린 뒤에는 다시 들어가기 어렵습니다. ON/OFF 떨림이 자연스럽게 사라집니다.

## 결과

세 가지를 합친 결과는 다음과 같이 정리됩니다.

| 상황 | 동작 |
|---|---|
| 손님이 잠깐 햇빛에 인상을 찌푸림 | EMA가 거의 흔들리지 않음 → 모객 계속 |
| 손님이 잠깐 옆을 봐서 인식 신뢰도 낮음 | 가중치 낮아 평균에 반영 안 됨 → 모객 계속 |
| 손님이 명백히 불쾌한 표정을 N 프레임 유지 | 가드 발동 → 모객 시퀀스 즉시 밀려남 |
| 손님 표정이 좋아져 EMA가 해제 임계 위로 회복 | 가드 해제 → 모객 재개 가능 |

체감으로는 **"잘못 멈추지 않으면서, 멈춰야 할 때는 즉시 멈춘다"**가 되었습니다. 모객 중단이 한 번 발생하면 한참 머물러 있어 손님이 충분히 반응할 시간을 확보하고, 회복도 사람 입장에서 자연스러운 속도로 일어났습니다.

## 배운 점

- 행동 트리의 **최상위 노드 선택**이 곧 반응성·안전성의 정책 선언이라는 점이 인상적이었습니다. `Sequence`/`Fallback`과 `ReactiveSequence`/`ReactiveFallback`은 한 글자 차이처럼 보여도, 위급한 상황에서의 동작이 완전히 다릅니다.
- 신호를 **트리에 넣기 전에 다듬는 단계**(EMA·신뢰도 가중)와 **트리 안에서 판단하는 단계**(히스테리시스)를 분리해 두면, 각각을 따로 튜닝할 수 있어 디버깅이 쉽습니다.
- HRI(인간-로봇 상호작용)에서는 **빠른 반응**과 **부드러운 반응**이 동시에 요구됩니다. 둘은 충돌하는 요구처럼 보이지만, "안전·부정 가드는 즉각, 일반 신호는 부드럽게"로 책임을 갈라 두면 충돌 없이 양쪽을 만족시킬 수 있습니다.
