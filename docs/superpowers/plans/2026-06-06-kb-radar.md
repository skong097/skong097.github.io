# KB Radar Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 기존 `/dashboard/` 페이지를 제거하고, 사전 설정한 키워드로 최신 논문·기사·영상·인물을 주 2회 수집·요약해 원본 링크와 함께 보여주는 KB Radar 페이지(`/kb-radar/`)를 구축한다.

**Architecture:** Hugo + PaperMod 정적 사이트. 기존 dashboard와 동일하게 `data/` 파일을 `site.Data ... | jsonify`로 페이지에 주입해 클라이언트 JS로 렌더링한다. 수집은 로컬 cron이 래퍼 스크립트를 통해 `claude -p`(WebSearch/WebFetch)를 실행해 `data/kb_items.json`만 갱신하고, 커밋은 사용자가 직접 한다.

**Tech Stack:** Hugo 0.162.1 (extended), PaperMod 테마, YAML/JSON 데이터, 순수 HTML/CSS/JS 레이아웃, bash 래퍼 + bash completion, Claude Code CLI.

> **커밋 정책:** 사용자 전역 규칙상 에이전트는 `git add/commit/push`를 실행하지 않는다. 각 Task의 "Commit" 스텝은 사용자에게 **복사-붙여넣기용 명령 블록을 제시**하는 것으로 처리하며, 실제 커밋은 사용자가 수행한다.

> **검증 방식:** 단위 테스트 프레임워크가 없으므로, TDD의 "실패하는 테스트 → 통과" 사이클을 정적 사이트에 맞게 **(1) 검증 명령으로 현재 실패/부재 확인 → (2) 구현 → (3) 동일 명령으로 통과 확인**으로 대응한다. 검증 도구: `hugo`(빌드), `grep`(산출 HTML 확인), `python3 -m json.tool`(JSON 유효성).

<script>
const KB = JSON.parse({{ (site.Data.kb_items) | jsonify }});
const KW = JSON.parse({{ (site.Data.kb_keywords) | jsonify }});
const ITEMS = (KB && KB.items) || [];
const GROUPS = (KW && KW.groups) || [];
const CAT_LABEL = { paper: "논문", article: "기사", video: "영상", person: "인물" };

const state = { cats: new Set(["paper","article","video","person"]), archived: false, q: "" };

document.getElementById("kbrLast").textContent = (KB && KB.last_collected) || "—";

function visible(it) {
  if (!state.cats.has(it.category)) return false;
  if (!state.archived && it.archived) return false;
  if (state.q) {
    const hay = ((it.title||"") + " " + (it.summary||"") + " " + (it.group||"")).toLowerCase();
    if (!hay.includes(state.q)) return false;
  }
  return true;
}

function cardHtml(it) {
  const cat = it.category || "article";
  const date = it.published || it.collected || "";
  const src = it.source || "";
  return `<div class="kbr-card${it.archived ? ' archived' : ''}">
    <span class="kbr-badge ${cat}">${CAT_LABEL[cat] || cat}</span>
    <div class="kbr-card-title"><a href="${it.url}" target="_blank" rel="noopener">${it.title || "(제목 없음)"}</a></div>
    <div class="kbr-summary">${it.summary || ""}</div>
    <div class="kbr-card-foot">
      <span class="kbr-source">${src}${date ? " &middot; " + date : ""}</span>
      <span>↗</span>
    </div>
  </div>`;
}

function render() {
  const main = document.getElementById("kbrMain");
  const shown = ITEMS.filter(visible);
  document.getElementById("kbrCount").textContent = shown.length;

  if (ITEMS.length === 0) {
    main.innerHTML = `<div class="kbr-empty">아직 수집된 항목이 없습니다.<br>
      <code>scripts/run_kb_radar.sh</code> 를 실행하면 항목이 채워집니다.</div>`;
    return;

  // 알려진 그룹 순서대로, 그 다음 미분류(orphan) 그룹
  const order = GROUPS.map(g => g.id);
  const nameOf = {}; GROUPS.forEach(g => nameOf[g.id] = g.name);
  const orphans = [...new Set(shown.map(i => i.group).filter(g => !order.includes(g)))];
  const allGroups = order.concat(orphans);

  let html = "";
  allGroups.forEach(gid => {
    const list = shown.filter(i => i.group === gid);
    if (list.length === 0) return;
    list.sort((a,b) => (b.published || b.collected || "").localeCompare(a.published || a.collected || ""));

      <h3 class="kbr-group-title">${nameOf[gid] || gid}
        <span class="kbr-group-count">${list.length}</span></h3>
      <div class="kbr-grid">${list.map(cardHtml).join("")}</div>
    </section>`;
  });
  main.innerHTML = html || `<div class="kbr-empty">필터 조건에 맞는 항목이 없습니다.</div>`;
}

document.querySelectorAll(".kbr-chip[data-cat]").forEach(chip => {

    if (state.cats.has(c)) { state.cats.delete(c); chip.classList.remove("on"); }
    else { state.cats.add(c); chip.classList.add("on"); }
    render();
  });
});

너는 이 저장소(`/home/gjkong/dev_ws/blog`)에서 KB Radar 데이터를 갱신하는 수집 에이전트다.

## 절차
1. 위 두 파일을 읽는다. `kb_items.json`이 없거나 비어 있으면 `{ "last_collected": null, "items": [] }`로 시작한다.

   - 최신성을 우선한다(가능하면 최근 30일 이내).
3. 유망한 후보는 **WebFetch**로 실제 내용을 확인한 뒤 채택한다(접근 불가/내용 빈약하면 버린다).
4. 채택 항목마다 아래 필드를 채운다:
   - `id`: 원본 URL의 SHA1 해시(소문자 hex). 중복 판정 키.
   - `group`: 해당 그룹 `id`.

   - `title`, `summary`(**한국어 2~3문장**, 사실 위주·과장 없이), `source`(매체/플랫폼명), `authors`(없으면 ""),

6. **그룹 상한**: 한 번 실행에서 그룹별 신규 항목은 `max_per_run`을 넘지 않는다.
7. **아카이브**: 기존 항목 중 `collected`가 오늘 기준 30일을 초과한 것은 `archived: true`로 바꾼다(삭제 금지).

9. **절대 git add/commit/push 하지 않는다.** 파일만 저장한다.

- 요약은 한국어. 추측·홍보성 표현 금지.
- 출력 JSON은 항상 유효해야 한다(저장 전 형식 확인).

```bash
#!/usr/bin/env bash
# run_kb_radar.sh — KB Radar 수집 런처
# 옵션:
#   -h, --help        사용법 출력
#   --dry-run         수집을 시뮬레이션하되 data/kb_items.json 저장은 생략
#   --group <id>      특정 키워드 그룹만 수집 (kb_keywords.yaml의 group id)
# 동작: repo 루트에서 claude -p 로 scripts/kb_radar/collect.md 지시문을 실행한다.
#       git 커밋은 하지 않는다(사용자가 직접). 변경 요약을 출력한다.
set -euo pipefail

REPO_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
COLLECT_MD="$REPO_ROOT/scripts/kb_radar/collect.md"
ITEMS_JSON="$REPO_ROOT/data/kb_items.json"

DRY_RUN=0
GROUP=""

usage() {
  sed -n '2,9p' "${BASH_SOURCE[0]}" | sed 's/^# \{0,1\}//'
}

while [[ $# -gt 0 ]]; do
  case "$1" in
    -h|--help) usage; exit 0 ;;
    --dry-run) DRY_RUN=1; shift ;;
    --group) GROUP="${2:-}"; shift 2 ;;
    --group=*) GROUP="${1#*=}"; shift ;;
    *) echo "알 수 없는 옵션: $1" >&2; usage; exit 2 ;;
  esac
done

if [[ ! -f "$COLLECT_MD" ]]; then
  echo "지시문이 없습니다: $COLLECT_MD" >&2; exit 1
fi

PROMPT="$(cat "$COLLECT_MD")"
if [[ -n "$GROUP" ]]; then
  PROMPT="$PROMPT

## 이번 실행 한정 지시
오직 group id 가 \"$GROUP\" 인 그룹만 수집한다. 나머지 그룹은 건너뛴다."
fi
if [[ "$DRY_RUN" -eq 1 ]]; then
  PROMPT="$PROMPT

## 이번 실행 한정 지시 (DRY RUN)
검색·요약까지만 수행하고, data/kb_items.json 파일은 절대 저장하지 마라. 무엇을 추가할지 요약만 출력하라."
fi

BEFORE_HASH="$( [[ -f "$ITEMS_JSON" ]] && md5sum "$ITEMS_JSON" | cut -d' ' -f1 || echo none )"

echo "[KB Radar] 수집 시작 (dry-run=$DRY_RUN, group=${GROUP:-all})"
cd "$REPO_ROOT"
claude -p "$PROMPT"

AFTER_HASH="$( [[ -f "$ITEMS_JSON" ]] && md5sum "$ITEMS_JSON" | cut -d' ' -f1 || echo none )"
if [[ "$BEFORE_HASH" != "$AFTER_HASH" ]]; then
  COUNT="$(python3 -c 'import json,sys; print(len(json.load(open(sys.argv[1]))["items"]))' "$ITEMS_JSON" 2>/dev/null || echo '?')"
  echo "[KB Radar] data/kb_items.json 변경됨 — 총 항목 $COUNT 개. 'git diff data/kb_items.json' 확인 후 커밋하세요."
else
  echo "[KB Radar] data/kb_items.json 변경 없음."
fi
```

# KB Radar bash 자동완성
# 부트스트랩: ~/.bashrc 에 아래 한 줄 추가
#   [ -f "$HOME/dev_ws/blog/scripts/completion.bash" ] && source "$HOME/dev_ws/blog/scripts/completion.bash"

_run_kb_radar() {
  local cur prev opts
  COMPREPLY=()
  cur="${COMP_WORDS[COMP_CWORD]}"
  prev="${COMP_WORDS[COMP_CWORD-1]}"
  opts="-h --help --dry-run --group"

  if [[ "$prev" == "--group" ]]; then
    # kb_keywords.yaml 의 group id 동적 추출
    local kwfile groups
    kwfile="$(dirname "${BASH_SOURCE[0]}")/../data/kb_keywords.yaml"
    if [[ -f "$kwfile" ]]; then
      groups="$(grep -E '^\s*-\s*id:' "$kwfile" | sed -E 's/.*id:\s*//; s/["'\'']//g')"
      COMPREPLY=( $(compgen -W "$groups" -- "$cur") )
    fi
    return 0
  fi

  COMPREPLY=( $(compgen -W "$opts" -- "$cur") )
}
